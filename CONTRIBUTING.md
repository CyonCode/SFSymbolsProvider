# Contributing to SFSymbolsProvider

First off, thank you for considering contributing to SFSymbolsProvider! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How to Contribute

### Reporting Bugs
- Check the [Issues](https://github.com/yourusername/SFSymbolsProvider/issues) to see if the bug has already been reported.
- If not, open a new issue. Provide a clear title and description, as much relevant information as possible, and a code sample or executable test case demonstrating the expected behavior that is not occurring.

### Suggesting Features
- Open a new issue and describe the feature you'd like to see, why it's useful, and how it should work.

### Pull Requests
1. Fork the repository and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`swift test`).
4. Make sure your code follows the existing style (Swift API Design Guidelines).
5. Issue that pull request!

## Code Style Guidelines
- Follow the [Swift API Design Guidelines](https://swift.org/documentation/api-design-guidelines/).
- Use 4 spaces for indentation.
- Keep methods and classes focused and concise.

## Testing Requirements
Before submitting a pull request, please run the tests to ensure everything is working as expected:
```bash
swift test
```
All tests must pass for a contribution to be accepted.
